# ハンドオフ: MyFamiList（家族・グループ買い物リスト共有アプリ）

## 概要 / Overview
MyFamiList は、家族やグループで買い物リストをリアルタイム共有する iOS アプリです。
既存の個人用アプリ「MyShopMemo」のグループ共有版として、別アプリとして App Store 公開予定。

本バンドルは、アプリの主要フローを **操作可能な HTML ハイファイ・プロトタイプ** として実装したものです。
- ログイン → リスト一覧（グループ切替）→ **リスト詳細（中核画面）** → アイテム編集
- メンバー一覧・招待 / アップグレード（買い切り）/ 設定
- 無料プラン上限の表示と、上限到達時のアップグレード誘導

**この中の HTML / JSX ファイルは「デザイン参照物」です。**

## 画面 / Screens

> 共通: デバイス上端のステータスバー/ダイナミックアイランドを避けるため、各画面ヘッダーは上 54px のクリアランスを取る。下部はホームインジケータ分（約 24〜34px）の余白。画面背景は選択テーマの `bg`。

### 1. ログイン / オンボーディング `LoginScreen`
- **目的**: 初回サインイン。
- **レイアウト**: 縦中央にロゴ、下部にサインインボタン3つ。背景に soft 色の大きな円を2つ装飾配置（淡い）。
- **コンポーネント**:
  - アプリアイコン: 84×84、角丸24、塗り = primary、中に白いカート（SF Symbol `cart.fill`）46pt。影 `0 16px 34px -10px <primary>`。
  - タイトル「MyFamiList」33pt/Bold。サブ「家族やグループの買い物リストを、みんなでリアルタイムに共有。」14〜16pt/textSec、中央寄せ、行間1.6。
  - ボタン（高さ54、角丸=ボタン半径、横幅100%、アイコン+ラベル、gap10）:
    1. **Appleでサインイン** — 背景 `#000`、文字白、SF Symbol `apple.logo`
    2. **Googleで続ける** — 背景 surface、1px sep 枠、Google多色ロゴ
    3. **メールアドレスで続ける** — 背景 surface、1px sep 枠、SF Symbol `envelope`
  - 最下部に規約注記 11.5pt/textTer。
- **挙動**: いずれのボタンでも認証完了 → リスト一覧へ。認証成功 1.3 秒後に「花子さんが『牛乳』を追加しました」トーストを上部から表示（リアルタイム同期の演出）。

### 2. リスト一覧 `ListsScreen`（タブ: リスト）
- **目的**: 現在のグループのリストを一覧、開く。グループ切替。
- **ヘッダー**:
  - 大タイトル左上に **グループ切替ピル**（surface 背景・影・角丸999、`絵文字 + グループ名 + ⌄`）。タップで GroupPicker シート。
  - 大タイトル「リスト」。右側にグループのメンバー **アバタースタック**（重なり表示、最大4＋"+n"）。タップでも GroupPicker。
- **本体**: リストカード（縦 gap 12）。各カード（surface・角丸カード・影・padding 15/16）:
  - 左: **進捗リング**（38×38, 太さ3.5, 未完了数を中央に数字表示, 進捗 = 完了数/総数, 色 = グループ色）
  - 中: リスト名 17.5pt/600、サブ「残り N品 ・ 全M品」（完了時は「完了 🎉」）13.5pt/textSec
  - 右: chevron（textTer）
  - カード下段（hairline 区切り）: 未完了アイテムのカテゴリ色ドット最大5個 + カテゴリ名の連結テキスト 12.5pt/textTer
  - タップでリスト詳細へ（右からスライドイン遷移）。
- **下**: 「＋ リストを追加」破線ボタン（角丸カード, dashed textTer）。
- **無料プラン時**: 最下部に「無料プラン ・ リスト N/2」12.5pt/textTer。
- **タブバー**: 下部にガラス風（blur）3タブ: リスト / メンバー / 設定。アクティブは primary 色。

### 3. リスト詳細 `ListDetailScreen` 【中核画面】
- **目的**: 商品のチェック・追加・編集。
- **ヘッダー**: 戻る chevron（primary）+ リスト名（大タイトル）。右に「…」メニュー丸ボタン（surface・影）。タイトル下に「絵文字 グループ名 ・ N品 未購入」。
- **本体**:
  - **未購入セクション**: surface カード内に行を並べる。各 `ItemRow`:
    - 左: **円形チェックボックス**（26px。未チェック=1.8px textTer 枠の空丸 / チェック=グループ色塗り+白✓。チェック時アニメーション）
    - カテゴリ色ドット（10px）
    - 商品名（16.5〜17pt/500）+ 数量（やや小・textSec, 左マージン8）
    - 2行目（条件付き）: メモがあれば「📝 メモ内容」、無ければ他人が追加した物に限り「○○さんが追加 ・ 8分前」12.5pt/textTer
    - 右: 自分以外が追加した未購入品にその人のアバター（22px）
    - タップ → アイテム編集シート。チェックボックスタップ → 完了へ移動。
  - 空状態: 🛒 + 「未購入の商品はありません」。
  - **「カゴに入れた (n)」セクション**: 折りたたみ可能（chevron 回転）。完了アイテムは取り消し線・textTer・カード全体 opacity 0.78。
- **下部固定コンポーザ**（ガラス風 blur）:
  - 折りたたみ時: 「＋ 商品を追加…」入力欄（fieldBg・角丸入力）。
  - フォーカスで展開: 上にカテゴリ選択チップ（横スクロール, 選択中はカテゴリ色塗り+白文字, 非選択は fieldBg+ドット）。右に「追加」ボタン（グループ色, 入力があるとき）/「閉じる」。
  - Enter または「追加」で未購入リストの先頭に追加、入力欄は維持しフォーカス継続（連続入力）。

### 4. メンバー `MembersScreen`（タブ: メンバー）
- **目的**: メンバー確認・招待。所属グループの可視化。
- **本体**: surface カードにメンバー行（上揃え, padding 13/16, hairline 区切り）:
  - アバター42px、氏名16.5pt/600（自分は「（あなた）」を textTer で付記）、役割13pt/textSec（先頭=オーナー、他=メンバー）
  - **所属グループチップ**（その人が属する全グループ）: ピル（絵文字+グループ名 12pt）。**現在のグループは soft 背景+softText でハイライト**、他は fieldBg+textSec。→ 複数グループ所属が一目で分かる。
  - オーナー行の右に王冠アイコン（softText）。
- **下**: 「＋ メンバーを招待」primary ボタン → 招待シート。下に「無料プランは N/3 人まで」（Pro は「無制限」）。

### 5. 設定 `SettingsScreen`（タブ: 設定）
- プロフィールカード（アバター50 + 氏名 + 「Apple ID でサインイン中」）。
- **無料プラン時**: 「MyFamiList Pro」グラデーション（primary→press）カード CTA → アップグレード。
- 「通知」グループ: プッシュ通知（オン）/ リマインダー（土 10:00）。「その他」: カテゴリの管理 / ヘルプ / サインアウト。各行はアイコン角丸タイル + ラベル + 値 + chevron。
- 最下部にバージョン「MyFamiList v1.0.0」。

### シート（ボトムシート, 上端角丸30, グラブハンドル, 背景に半透明黒オーバーレイ）

**GroupPicker（グループ切替）**: グループ一覧（現在のものは soft 背景+primary 枠+✓）。各行: 絵文字タイル + 名前 + 「N個のリスト ・ M人」+ アバタースタック。下に「＋ 新しいグループ」破線（無料時は王冠アイコン付）。

**EditItem（アイテム編集）**: 上部に「キャンセル / 商品を編集 / 保存」。商品名入力 / 数量入力（右寄せ）/ カテゴリチップ（折返しグリッド, 選択=カテゴリ色）/ メモ（textarea）/ 「削除」ボタン（淡い赤 `#FBEAE8` 背景・赤文字 `#D9695F`）。

**Invite（招待）**: people アイコン丸 + 「『グループ名』に招待」+ 説明（未インストールは App Store へ自動案内）。QR コード（プレースホルダ）+ 招待コード `FAMI-XXXX-XXX`（等幅）。リンク行 `myfami.app/j/...` + コピー（押すと「コピー済」✓）。「リンクを共有」primary。無料で3人上限到達時は「Pro にアップグレード」CTA に切替。

**Paywall（アップグレード, 買い切り）**: 王冠グラデアイコン + 「MyFamiList Pro」25pt/800 + 「一度の購入で、ずっと無制限。」。機能リスト4項目（グループ無制限 / リスト＆メンバー無制限 / 通知とリマインダー / 履歴と統計、各 soft タイルアイコン+✓）。価格「¥1,200 買い切り ・ 追加課金なし」。「購入して無制限にする」primary。下に「購入を復元 / 利用規約」。右上に×閉じる。

## インタラクションと挙動 / Interactions & Behavior
- **画面遷移**: タブ切替（即時）。リスト詳細は右からスライドイン（約 0.34s, ease-out）/ 戻るでスライドアウト。
- **シート**: 下からスライドアップ（約 0.34s）、背景タップまたはハンドルで閉じる。
- **押下フィードバック**: タップ可能要素は押下時に scale 0.97 前後 + 微小フェード（0.12s）。
- **チェック**: チェックボックスはトグルで塗り/枠＋✓がスプリング的に出現。完了品は「カゴに入れた」へ移動。
- **連続追加**: コンポーザは追加後も開いたままフォーカス維持。
- **トースト**: ログイン後の擬似リアルタイム通知（上部スライドイン、約3.6秒で自動消去）。製品では Firestore のリアルタイム更新を反映。
- **無料プラン上限**: 下記参照。上限到達アクションは Paywall を提示。

## 状態管理 / State Management（SwiftUI @Observable 想定）
プロトタイプの React state を、製品では `@Observable` モデル + Firestore に対応づけてください。

- **認証状態** `isLoggedIn`（Firebase Auth のセッション）
- **プラン** `plan: free | pro`（StoreKit 2 の購入状態・買い切りエンタイトルメント）
- **グループ配列** `groups`（Firestore: `groups` コレクション。各 group が members[], lists[]）
- **現在のグループ** `currentGroupId`
- **タブ** `tab: lists | members | settings`
- **開いているリスト** `openListId`（詳細プッシュ）
- **編集中アイテム** `editingItemId`
- **アクティブなシート** `sheet: group | invite | paywall | nil`
- **トースト** `toast`

データ更新（チェック/追加/編集/削除）は Firestore へ書き込み、リアルタイムリスナーで全メンバーに即時反映。

### データモデル
```
Member  { id, name(表示名/イニシャル用), full(氏名), color(アバター色) }
Group   { id, name, emoji, color, members:[memberId], lists:[List] }
List    { id, name, note, items:[Item] }
Item    { id, name, qty(文字列: "2本"/"300g"等), cat(カテゴリ名),
          checked:Bool, memo, by(memberId), at(追加時刻表示) }
招待     { invite code + URL（myfami.app/j/<code>）, 未インストールは App Store へ誘導 }
```

### 無料プラン上限（重要なビジネスロジック）
- 無料: **作成グループ1 ・ リスト2/グループ ・ メンバー3/グループ**
- 買い切り（¥1,200）後: すべて無制限
- 上限到達時の操作（新規グループ作成 / 3つ目のリスト追加 / 4人目の招待）→ **Paywall を提示**
- 注: 他人のグループへ「参加」は作成数上限に含めない想定（プロトタイプでは自分作成の「山田家」＋招待参加の「職場の買い出し」の2所属を表示）
